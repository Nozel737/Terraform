# 🚀 Terraform Basics

Terraform uses a declarative approach, meaning you describe the desired infrastructure, and Terraform determines the actions required to achieve that state.

Terraform configurations are written using **HashiCorp Configuration Language (HCL)**, a simple and human-readable language designed specifically for defining infrastructure.

Every Terraform project follows a standard workflow:

## 1. Write Configuration

Define infrastructure using `.tf` files.

## 2. Initialize Terraform

```bash
terraform init


## 3. Validate Configuration
terraform validate

Checks the syntax and configuration before deployment.

## 4. Format Files
terraform fmt

Automatically formats Terraform code according to best practices.

## 5. Review Execution Plan
terraform plan

Displays the actions Terraform intends to perform without modifying the infrastructure.

## 6. Apply Changes
terraform apply

Creates or updates infrastructure based on the configuration.

## 7. Destroy Infrastructure
terraform destroy

Removes all managed resources safely.
