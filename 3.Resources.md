# ⚙️ Resources

## Resources are the fundamental building blocks of every Terraform configuration.

A resource represents any infrastructure object managed by Terraform, such as:

- Virtual Machines
- Networks
- Security Groups
- Databases
- Storage Buckets
- Load Balancers
- IAM Users
- DNS Records

## Example:

```terraform
resource "aws_instance" "web_server" {
  ami           = "ami-123456789"
  instance_type = "t2.micro"
}

## A resource consists of:

Resource Type
Resource Name
Configuration Arguments

Terraform tracks every resource inside the state file, allowing it to determine which resources should be created, updated, or destroyed during future executions.

Resources can reference one another, making it possible to build complex infrastructure with automatic dependency management.
