# 🌎 Workspaces & Environments

Applications typically run in multiple environments, each serving a different purpose.

Common environments include:

- Development (Dev)
- Testing (Test)
- Staging
- Production (Prod)

Terraform Workspaces allow engineers to manage multiple environments using the same configuration while maintaining separate state files.

Create a workspace:

```bash
terraform workspace new development
```

List available workspaces:

```bash
terraform workspace list
```

Switch workspace:

```bash
terraform workspace select production
```

Each workspace maintains its own isolated state, allowing infrastructure changes to remain independent across environments.

Benefits of workspaces:

- Environment isolation
- Easier testing
- Reduced code duplication
- Simplified infrastructure management

Although workspaces are useful, large production deployments often combine workspaces with separate directories or reusable modules to improve maintainability.
