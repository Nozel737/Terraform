# 📂 Terraform State

Terraform keeps track of every managed resource using a **state file**.

The default state file is:

```text
terraform.tfstate
```

The state file acts as Terraform's source of truth, storing metadata about every managed resource.

Terraform uses the state file to:

- Track existing infrastructure
- Detect configuration drift
- Determine required changes
- Improve execution performance
- Resolve dependencies

There are two types of state:

## Local State

Stored on the local machine.

Suitable for learning and personal projects.

## Remote State

Stored in a shared backend such as:

- AWS S3
- Azure Storage
- Google Cloud Storage
- Terraform Cloud

Remote state is recommended for production because it enables collaboration, state locking, backup, and centralized management.
